## What is software?

```bash
Computer programs and associated documentation.
```

## Why use software engineering to develop a software?
```bash
To reduce costs of maintenance. To make a software virtually possible to maintain at all.
```

## Attributes of good software?

```bash
📍 functionality
📍 performance
📍 maintainability
📍 dependability
📍 usability
📍 efficiency
📍 dependability and security
📍 acceptability
📍 robustness
📍 trustworthy... etc
📍 You cannot be precise with software.
```

## Key challenges?

```bash
Coping with increasing diversity, demands for reduced times and developing trustworthy software.
```

## Costs?

```bash
Roughly 60% of sw costs are development, 40% are assets, not mentioning maintenance costs.
```

## Software process activities:
```bash
📍 Software specification - input to legal contract. Defined by customers and engineers.

📍 Software development - where the software is designed and programmed.

📍 Software validation - validation passes upon matching specification.

📍 Software evolution - reflects evolution of specification.
```

## Issues that affect software:
```bash
📍 Heterogeneity

📍 Security and trust

📍 Scale
```

## Application types:
```bash
📍 Batch procesing systems. Systems designed to process data in large batches at once while stopped.
📍 etc
```
